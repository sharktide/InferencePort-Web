    Subscription API (Generation)
    
          
    
            
    The Generation API (prefix /gen) provides AI generation backed by your plan's subscription quota. Each subscription tier (Free, Light, Core, Creator, Professional) includes daily/weekly usage limits.
    
            
    Endpoints
    
            
      
              
    • POST /gen/chat/completions — Text chat completions
      • 
              
    • POST /gen/images/generations — Image generation
      • 
              
    • POST /gen/videos/generations — Video generation
      • 
              
    • POST /gen/audio/generations — Audio/music generation
      • 
              
    • POST /gen/3d/generations — 3D generation (P2G API only)
      • 
              
    • GET /gen/models — Available model listing
      • 
            
    
            
    Authentication
    
            
    Use Authorization: Bearer <your-supabase-jwt> or Authorization: Bearer <lightning-api-key>.
    
            
    Rate Limits
    
            
    Rate limits are determined by your subscription tier. See your plan for specific daily/weekly quotas.

    Pay-2-Go (P2G) API
    
          
    
            
    The Pay-2-Go API (prefix /v1) provides credit-billed AI generation. Purchase credits and consume them per request. No monthly subscription required — only pay for what you use.
    
            
    Endpoints
    
            
      
              
    • POST /v1/chat/completions — Text chat completions
      • 
              
    • POST /v1/images/generations — Image generation
      • 
              
    • POST /v1/videos/generations — Video generation
      • 
              
    • POST /v1/audio/generations — Audio/music generation
      • 
              
    • POST /v1/3d/generations — 3D generation (image-to-3d)
      • 
              
    • GET /v1/models — Available models
      • 
              
    • GET /v1/me — Account & wallet info
      • 
              
    • GET /v1/credits/ledger — Credit transaction history
      • 
              
    • GET /POST /v1/lightning-api-keys — API key management
      • 
            
    
            
    Authentication
    
            
    Use Authorization: Bearer <your-supabase-jwt> or Authorization: Bearer <lightning-api-key>.
    
            
    Pricing
    
            
    Credits are consumed per request. View your Account tab for current credit balance and purchase packs. See the Models tab for per-model pricing.

    AI Shield
    
          
    
            
    AI Shield is an intelligent abuse prevention and fraud detection system. It analyzes requests in real time to detect and prevent abuse, fraud, and malicious usage of your AI applications.
    
            
    How It Works
    
            
    Shield employs a multi-layered analysis pipeline:
    
            
      
              
    1. Signal Collection — Gathers email, IP, phone, device, content, and behavioral signals from each request.
      2. 
              
    2. Heuristic Analysis — Runs fast pattern-based checks for prompt injection, spam, and known abuse patterns.
      3. 
              
    3. Intelligence Collection — Queries external and internal intelligence sources for IP reputation, email risk, device fingerprinting, and more.
      4. 
              
    4. Agent Analysis — Specialized agents analyze identity fraud, exfiltration attempts, prompt threats, content policy violations, and payment fraud.
      5. 
              
    5. Relationship Graph — Maps entity relationships to detect duplicate accounts and coordinated campaigns.
      6. 
              
    6. Historical Memory — Checks encrypted global and customer-specific memory for prior abuse history.
      7. 
              
    7. LLM Reasoning — Synthesizes all evidence through an LLM for final risk scoring and decision.
      8. 
              
    8. Decision Engine — Produces a final verdict: allow, challenge, rate_limit, review, or block.
      9. 
            
    
            
    Endpoint
    
            
    POST /ai-shield/analyze — Submit signals for abuse analysis.
    
            
    Request Body
    
            
    {
  "email": "user@example.com",
  "phone": "+1234567890",
  "username": "johndoe",
  "ip": "203.0.113.1",
  "device_fingerprint": "abc123...",
  "content": "...",
  "geolocation": { ... },
  "signup_time": "2025-01-01T00:00:00Z",
  "metadata": { ... }
}
    
            
    Response Fields
    
            
      
              
    • risk_score — 0-100 overall risk
      • 
              
    • confidence — 0-1 confidence in the decision
      • 
              
    • decision — allow, challenge, rate_limit, review, or block
      • 
              
    • recommended_action — Human-readable recommended action
      • 
              
    • threat_categories — Identified threat types
      • 
              
    • reasons — Human-readable explanations
      • 
              
    • duplicate_user_score — Likelihood this is a duplicate account (0-100)
      • 
              
    • linked_accounts — Number of accounts linked via shared entities
      • 
              
    • campaign_risk_score — Coordinated campaign risk (0-100)
      • 
              
    • evidence — Detailed evidence items with source, risk, weight, confidence
      • 
              
    • investigation — Tools used, evidence count, campaign info
      • 
              
    • config_applied — The feature config used for this analysis
      • 
            
    
            
    Feature Config
    
            
    Include a config.features object to enable or disable specific analysis modules. All features default to true. Use the toggles in the playground below to experiment.
    
            
      
              
    • Signal: heuristics
      • 
              
    • Memory: historical_memory
      • 
              
    • Graph: duplicate_detection, campaign_detection
      • 
              
    • Intelligence: email_intelligence, ip_intelligence, phone_intelligence, username_intelligence, device_intelligence, behavior_intelligence, content_intelligence, prompt_intelligence
      • 
              
    • Agent: identity_analysis, fraud_analysis, prompt_analysis, content_analysis, exfiltration_analysis
      • 
              
    • Decision: llm_reasoning
      • 
              
    • Storage: memory_update
      •
